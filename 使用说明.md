## 🎯 按工作流选择任务

### 第1步: 需求阶段
**→ T01: Requirement Clarification**
- 澄清业务需求
- 确认范围和边界
- 识别风险
- 评估可行性

**Prompt开始语:**
```
我们需要澄清一个需求：[需求描述]

请帮我明确以下方面：
1. 核心目标是什么？
2. 哪些场景需要支持？
3. 性能要求？
4. 成功标准是什么？
```

---

### 第2步: 设计阶段
**→ T08: Architecture Decision Draft**
- 系统架构设计
- 技术栈选型
- 高层组件设计

**Prompt开始语:**
```
我需要为[功能]设计架构。

要求：
- 支持[规模描述]
- 约束：[技术/成本/时间约束]

请建议：
1. 整体架构
2. 技术栈
3. 数据存储方案
4. 潜在风险
```

**→ T09: API Contract Design**
- API设计和定义
- 数据模型设计
- 接口规范

**Prompt开始语:**
```

- [端点1及操作]
- [端点2及操作]

要求：
- 认证方式：[方式]
- 错误处理：[标准]
- 版本管理：[策略]
```

## 🎯 立即开始

**第一步：识别你的问题**
- 你面临什么编程挑战？
- 查看上面的"按问题类型快速查询"

**第二步：打开对应任务**
- 进入对应的T0X文件夹
- 打开README.md

**第三步：选择合适的Prompt**
- 查看示例Prompt
- 选择最相近的
- 复制到AI工具

**第四步：调整和使用**
- 根据你的具体情况调整
- 参考最佳实践和高级技巧
- 使用AI工具协助完成任务

---

## 💬 获得最佳结果的提示

1. **提供充分的背景** - 更多的上下文 = 更好的结果
2. **明确要求** - 具体说明你想要什么
3. **给出约束** - 时间、资源、性能等限制
4. **提供示例** - 展示期望的输入和输出
5. **迭代改进** - 第一次可能不完美，继续改进
