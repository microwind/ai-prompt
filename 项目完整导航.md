## 📊 按开发阶段的学习路径

### 🎯 阶段1: 需求与规划
- **T01: Requirement Clarification** - 澄清需求，确认范围
  - 📖 何时用：项目启动、需求评审
  - 💡 关键技能：需求分析、范围管理、风险识别

- **T08: Architecture Decision Draft** - 设计架构，做技术选型
  - 📖 何时用：系统设计、技术栈选择
  - 💡 关键技能：架构设计、权衡分析、决策记录

### 💻 阶段2: 设计与开发
- **T09: API Contract Design** - 设计API接口
  - 📖 何时用：模块接口设计、API定义
  - 💡 关键技能：接口设计、规范制定、文档生成

- **T02: Feature Implementation** - 编写功能代码
  - 📖 何时用：日常编码、功能开发
  - 💡 关键技能：代码生成、算法设计、错误处理

- **T04: Refactor with Behavior Safety** - 安全改进代码
  - 📖 何时用：代码改进、技术债处理
  - 💡 关键技能：代码优化、重构策略、风险控制

### 🐛 阶段3: 测试与质量
- **T05: Unit Test Generation** - 编写单元测试
  - 📖 何时用：测试补充、覆盖率提升
  - 💡 关键技能：测试设计、边界覆盖、Mock使用

- **T06: Integration Test Plan** - 规划集成测试
  - 📖 何时用：系统验证、上线前检验
  - 💡 关键技能：测试场景设计、系统集成验证

- **T07: Code Review Assistant** - 进行代码审查
  - 📖 何时用：PR审查、质量把控
  - 💡 关键技能：质量评估、最佳实践识别、建设性反馈

- **T03: Bug Reproduction and Root Cause** - 诊断和排查Bug
  - 📖 何时用：问题排查、性能诊断
  - 💡 关键技能：问题诊断、日志分析、根因追踪

### 🚀 阶段4: 优化与维护
- **T10: SQL/Query Optimization** - 优化数据库查询
  - 📖 何时用：性能优化、数据库调优
  - 💡 关键技能：查询分析、索引设计、性能优化

- **T12: Performance Optimization** - 整体性能优化
  - 📖 何时用：性能调优、资源优化
  - 💡 关键技能：性能分析、瓶颈识别、优化实施

- **T11: Security Review** - 进行安全审查
  - 📖 何时用：安全评估、漏洞检查
  - 💡 关键技能：漏洞识别、防护设计、安全加固

- **T13: Legacy Code Understanding** - 理解遗留代码
  - 📖 何时用：知识转移、技术债清理
  - 💡 关键技能：代码理解、知识转移、重构规划

### 📚 阶段5: 文档与交付
- **T14: Documentation Generation** - 生成项目文档
  - 📖 何时用：文档生成、知识沉淀
  - 💡 关键技能：文档组织、清晰表达、自动化生成

## 💾 每个任务通常包含什么

每个T0X文件夹都应该包含：

1. **README.md** ✅
   - 核心概念和理论
   - 最佳实践原则
   - 常见误区分析
   - 相关工具和资源

2. **examples.md** (即将补充)
   - 5-10个真实场景的Prompt示例
   - 每个示例都有预期输出
   - 包含不同的场景和复杂度

3. **checklist.md** 或 **best-practices.md** (即将补充)
   - 该任务的检查清单
   - 快速参考表格
   - 常用工具和命令

4. **templates.md** (可选，即将补充)
   - 快速使用的Prompt模板
   - 适配不同场景的变体
