## 内容生成类

### 影视推荐语和标签

#### 场景背景

在视频平台上，每部电影或电视剧都需要：
- **一句话推荐**: 放在海报下方，吸引用户点击（30-50字）
- **标签**: 用于分类、搜索、推荐（3-5个关键词）

#### 需求分析

```
核心挑战：
1. 推荐语要吸引目标受众，但不能剧透
2. 标签要准确反映内容特征
3. 需要考虑不同平台和语言
4. 风格要匹配平台的整体调性
```

---

#### ❌ 反例1: 模糊且无约束

```
请为《漫威: 复仇者联盟》生成推荐文案和标签。
```

**问题**:
- 没有说明推荐语的长度
- 没有说明风格（正经 vs 戏谑）
- 没有说明目标受众
- 没有说明要避免剧透

---

#### ✅ 正例1: 清晰的内容生成Prompt

```markdown
# 影视推荐语和标签生成

## 电影信息
**电影名**: 《流浪地球2》
**类型**: 科幻/灾难
**导演**: 张艺谋
**主演**: 吴京, 李雪健, 沈腾
**简介**: 在人类移动地球解决太阳膨胀危机的过程中，一场全新的灾难却悄悄降临...

## 推荐语要求
1. **长度**: 35-45字（含标点）
2. **风格**: 专业、引人入胜、不剧透
3. **目标受众**: 18-50岁的科幻爱好者
4. **关键卖点**:
   - 宏大的科幻设定
   - 中国特色的灾难题材
   - 视觉特效
   - 演员阵容
5. **禁止内容**:
   - 不能透露关键情节转折
   - 不能透露结局
   - 避免过度承诺

## 标签要求
1. **数量**: 5个标签
2. **类型**:
   - 2个类型标签（如：科幻、灾难）
   - 2个特征标签（如：宏大场景、视觉特效）
   - 1个情感标签（如：震撼、感人）
3. **风格**: 简洁、容易搜索、易于分类
4. **格式**: 中文，每个标签2-4字

## 输出格式
```json
{
  "推荐语": "[35-45字的推荐文案]",
  "标签": ["标签1", "标签2", "标签3", "标签4", "标签5"],
  "说明": "解释推荐语的设计思路和标签的选择理由"
}
```

## 参考例子
**推荐语**: "地球能否逃离末日？壮阔的科幻冒险，宏大的视觉盛宴，在绝望的黑暗中闪烁人性的光辉。"
**标签**: ["科幻", "灾难", "中国特色", "视觉特效", "震撼"]
```

**为什么更好**:
- 明确的字数限制
- 清晰的风格和目标受众
- 列出禁止的内容
- 明确的标签分类
- 提供了参考例子
- 明确的输出格式

```markdown
# 批量生成影视推荐语和标签

## 任务
为下列5部电影生成推荐语和标签

## 电影列表

### 1. 《甲方乙方》
**类型**: 喜剧
**年代**: 1997年
**导演**: 冯小刚
**简介**: 一个神秘的客人来到租赁公司，要求实现各种梦想...

### 2. 《新龙门客栈》
**类型**: 动作/武侠
**年代**: 1992年
**导演**: 徐克
**简介**: 在龙门客栈，一场关于身份、忠诚和命运的对决正在上演...

[... 其他3部电影 ...]

## 统一要求
1. **推荐语**:
   - 长度: 35-45字
   - 风格: 符合电影的年代和类型
   - 目标受众: 影视爱好者，年龄18-60

2. **标签**:
   - 数量: 5个
   - 类型: 2个类型 + 2个特征 + 1个情感

3. **特殊考虑**:
   - 老电影要突出其经典地位和文化价值
   - 要考虑现代观众的审美

## 输出格式
为每部电影生成JSON，然后汇总到一个数组中：
```json
[
  {
    "电影名": "甲方乙方",
    "推荐语": "...",
    "标签": [...]
  },
  ...
]
```

## 质量要求
1. 推荐语必须符合字数限制
2. 标签必须是有效的分类词汇
3. 每部电影的推荐语和标签要有差异化，不能类似
4. 没有语法错误或歧义
```

**为什么更好**:
- 支持批量生成，提高效率
- 明确的统一要求
- 指出了特殊考虑（老电影）
- 清晰的输出格式

---

#### 🎯 最佳实践建议

**调优方向**:
1. 如果推荐语太通用，可以加上"关键卖点具体是什么"
2. 如果标签不够准确，可以提供"标签库"（预定义的有效标签）
3. 如果需要A/B测试，可以要求生成2-3个版本

**扩展Prompt**:
```markdown
## 多版本生成（用于A/B测试）

请为同一部电影生成3个版本的推荐语：
- 版本A: 强调视觉特效和动作
- 版本B: 强调演员和人物关系
- 版本C: 强调故事和情感

对每个版本说明其针对的目标受众。
```

### 书籍简介生成

#### 场景背景

电商平台需要为书籍生成：
- **内容简介**: 吸引读者，体现书籍的核心价值（80-120字）
- **目标读者**: 这本书适合谁
- **关键概念**: 3-5个核心概念

---

#### ✅ 正例: 完整的书籍简介生成Prompt

```markdown
# 书籍简介生成

## 书籍信息
**书名**: 《深度工作》
**作者**: 卡尔·纽波特（Cal Newport）
**出版社**: 中国人民大学出版社
**出版年**: 2016年
**页数**: 296页
**原书名**: Deep Work: Rules for Focused Success in a Distracted World

## 核心内容摘要
本书分析了在信息爆炸时代如何进行深度工作。作者区分了"深度工作"和"浅层工作"，提出了4条规则来实现深度工作。书中通过多个案例展示了深度工作的力量和方法。

## 简介要求

### 内容简介 (80-120字)
要求：
1. 第一句话点明核心问题或机会
2. 第二段简述解决方案或框架
3. 最后一句话总结收益或行动号召
4. 语言要简洁有力，避免废话
5. 要能激发读者的兴趣

示例风格：
"在信息时代，如何保持专注和提高工作效率？本书提出'深度工作'的概念...
通过4条规则...
如果你想在浮躁的时代脱颖而出，这本书必读。"

### 目标读者 (3-5个具体描述)
描述谁会从这本书中受益：
- 知识工作者（程序员、设计师、写手等）
- 学生（需要专注学习）
- 管理者（团队效率问题）
- [其他群体]

每个群体用1句话说明收益：
"对程序员来说：将提高代码质量和开发效率。"

### 关键概念 (3-5个核心概念)
从书中提取或总结的核心概念，格式：
"概念名：简短说明（一句话）"

例如：
- 深度工作：需要专注和无干扰的工作状态
- 浅层工作：容易分心且低价值的活动
- ...

## 输出格式

```json
{
  "书名": "深度工作",
  "内容简介": "[80-120字的简介]",
  "目标读者": [
    {
      "群体": "知识工作者",
      "收益": "..."
    },
    ...
  ],
  "关键概念": [
    "概念1：说明1",
    "概念2：说明2",
    ...
  ],
  "销售卖点": "一句话概括为什么要买这本书"
}
```

## 质量标准
1. 简介必须符合字数限制（80-120字）
2. 目标读者要具体，不能太宽泛
3. 关键概念要能代表整本书的核心
4. 没有错别字或语法错误
5. 简介要能吸引目标读者

## 参考案例
**简介**: "在信息爆炸的时代，深度工作成为稀缺且强大的技能。本书通过科学的案例和实践规则，教你如何建立深度工作的习惯，从而在职业生涯中取得突出成就。如果你想在竞争中脱颖而出，这是必读之书。"

**目标读者**:
- 程序员/设计师：提高专注力和创意质量
- 学生：改善学习效率和成绩
- 企业管理者：建立高效的团队文化

**关键概念**:
- 深度工作：在没有干扰的状态下，最大化智力能力的工作
- 目标堆栈：优先级和时间管理的方法
- 空闲时间的价值：休息如何支持深度工作
```

### 商品文案生成

#### ✅ 正例: 电商商品文案Prompt

```markdown
# 电商商品文案生成

## 商品信息
**商品名**: 智能运动手表 - FitPro Max
**品类**: 可穿戴设备 / 健身追踪器
**价格**: ¥1,299（预期）
**核心功能**:
- 心率监测（24/7）
- GPS定位（±5米精度）
- 50+ 运动模式
- 防水等级：5ATM
- 续航：14天
- 屏幕：AMOLED, 1.4英寸

**目标用户**: 25-45岁，重视健身和健康的都市白领
**竞争对手**: Apple Watch, Garmin, 华为

## 文案要求

### 1. 核心卖点（一句话）
**要求**:
- 15-25字
- 直击痛点或需求
- 包含最重要的功能优势

**示例风格**：
"精准心率 + 全能运动模式，让你成为自己的健身教练"

### 2. 标题文案
**要求**:
- 长度：25-50字
- 结构：功能 + 情感/利益
- 能在手机端清晰展示

**示例**:
"14天续航 + 50+运动模式，精准监测每一次心跳，智能手表该有的样子"

### 3. 详细描述（2-3段，200-300字总计）

**第1段** - 问题和场景：
- 描述目标用户的痛点
- 创造场景感

**第2段** - 解决方案：
- 介绍商品如何解决问题
- 突出2-3个核心功能
- 说明技术优势

**第3段** - 利益和行动：
- 使用这个商品会得到什么
- 行动号召（下单、加购等）

### 4. 特色卖点列表
**格式**: 3-5个核心卖点，每个用一句话 (15-25字)

示例:
- ✓ 精准心率：医疗级传感器，误差<5bpm，24/7全天监测
- ✓ 超长续航：14天电池续航，一周一充成为过去
- ✓ 运动全能：50+运动模式，覆盖你的每一项运动
- ✓ 防水防尘：5ATM防水等级，游泳冲浪无压力
- ✓ 智能推荐：AI根据你的运动数据给出个性化训练建议

## 输出格式

```json
{
  "核心卖点": "一句话卖点",
  "标题": "标题文案",
  "详细描述": {
    "第1段": "问题和场景...",
    "第2段": "解决方案...",
    "第3段": "利益和行动..."
  },
  "特色卖点": [
    "卖点1",
    "卖点2",
    ...
  ],
  "适配人群": "这个商品适合...",
  "常见疑问": [
    {
      "问": "续航真的有14天吗？",
      "答": "..."
    }
  ]
}
```

## 质量检查清单
- [ ] 没有虚假或夸大宣传
- [ ] 技术参数准确
- [ ] 避免使用过度营销的词汇（"绝对最强"等）
- [ ] 针对目标用户的痛点
- [ ] 文案简洁有力，没有冗长的句子
- [ ] 包含具体的利益点，而不是抽象的承诺
```

### 社交媒体文案

#### ✅ 正例: 小红书笔记文案Prompt

```markdown
# 小红书笔记文案生成

## 内容主题
**主题**: 分享一个"工作10年的程序员的学习心得"
**目标**: 获得赞和转发，建立专业形象
**产品推荐**: 一门编程课程（隐性）

## 核心信息
- 程序员容易陷入的学习误区
- 有效的学习方法
- 个人的成长故事

## 文案要求

### 标题 (15-30字)
要求：
- 包含数字或强词汇（"10年"、"踩坑"等）
- 引起目标受众的共鸣
- 有curiosity gap

示例:
"工作10年程序员的学习心得：我为什么不再"听起来学得很快""

### 正文 (150-300字)
结构：
1. 开头 (30-50字):
   - 讲一个具体的场景或反面例子
   - 引出问题

2. 中间 (80-150字):
   - 我之前的误区 (30字)
   - 后来的改进方法 (50-80字)
   - 具体的例子或数据 (20-40字)

3. 结尾 (20-50字):
   - 总结核心建议
   - 行动号召（点赞、关注、评论）

### 亮点部分
- 用emoji增加视觉吸引力
- 用加粗/斜体强调关键词
- 分段清晰，避免大块文字

## 示例结构

```
📌 标题在这里

开头场景：[具体故事]

🚫 我以前的问题：
[说出痛点]

✅ 后来的改进：
[详细解决方案]

💡 核心心得：
[1-3个关键学习]

点赞 + 关注，不迷路！
```

## 话题和标签建议
- 主话题: #程序员日常 #学习方法
- 相关话题: #职业成长 #技术分享
- 吸引话题: #干货分享 #工作10年的秘密

## 发布时间建议
工作日18:00-20:00，周末10:00-12:00
```

### 推荐系统设计

#### ✅ 正例: 内容推荐系统架构Prompt

```markdown
# 内容推荐系统设计

## 业务背景
**平台**: 视频内容社区（类似抖音/小红书）
**用户规模**: 500万日活用户
**内容库**: 100万+短视频
**核心指标**:
- CTR (点击率) 目标：> 8%
- 用户留存 (Day 7) 目标：> 35%

## 当前问题
- 推荐内容过于单一（用户看到相似内容）
- 新用户冷启动问题（无历史数据）
- 某些内容类型无法被发现（长尾内容）
- 推荐多样性不足

## 设计要求

### 1. 推荐算法框架
请设计一个混合推荐系统，包括：
- **协同过滤** (40%权重)
  - 用户相似度计算方法
  - 如何处理稀疏矩阵

- **内容分析** (30%权重)
  - 视频特征提取（标签、分类、时长等）
  - 特征向量化方法

- **热度和新鲜度** (20%权重)
  - 如何平衡热门内容和新内容
  - 衰减函数设计

- **多样性** (10%权重)
  - 推荐列表的多样性设计
  - 避免过度推荐单一话题

### 2. 冷启动问题解决
针对3种场景：
- 新用户冷启动：推荐策略
- 新内容冷启动：如何确保新视频有展示机会
- 新话题/创作者：如何帮助他们获得流量

### 3. 数据收集和更新
- 需要收集哪些数据？（用户行为、内容特征等）
- 推荐模型的更新频率？
- 如何处理用户隐私问题？

### 4. 评估指标
提供3-5个指标来评估推荐系统的效果：
- 用户相关指标（CTR、停留时间等）
- 内容相关指标（覆盖率、新内容曝光等）
- 业务指标（用户留存、LTV等）

## 输出格式

```markdown
# 推荐系统设计方案

## 整体架构
[架构图说明]

## 算法详解
### 1. 协同过滤
[详细设计]

### 2. 内容分析
[详细设计]

... [其他算法]

## 冷启动方案
### 新用户冷启动
[具体策略]

### 新内容冷启动
[具体策略]

### 新话题冷启动
[具体策略]

## 实现建议
- 技术栈：[推荐的技术]
- 数据流：[数据流向]
- 性能考虑：[如何优化]

## 评估指标体系
[详细指标定义]

## 可能的问题和解决方案
[常见问题和应对方案]
```

## 额外考虑
- 多国家/地区的推荐差异
- 不同设备（手机/PC）的推荐差异
- 推荐结果的透明度（用户为什么看到这个）
