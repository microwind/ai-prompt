# 01 代码生成基础

> 初级 ⭐ | 适合完全初学者快速生成代码片段

## 核心概念

代码生成是AI在编程中最常见的应用。好的代码生成Prompt应该：
- **明确需求**: 说清楚要做什么，不要模糊
- **指定语言**: 明确编程语言、版本、框架
- **给出上下文**: 说明代码的用途，不只是"写个函数"
- **明确约束**: 性能要求、依赖限制等
- **示例输入输出**: 如果可能，给出具体例子

## 最佳实践

🎯 **原则1: 具体说明需求**
- ❌ 差: "写个排序函数"
- ✅ 好: "用Python写一个函数，对整数列表进行升序排列，要求时间复杂度不超过O(nlogn)"

🎯 **原则2: 提供充分上下文**
- ❌ 差: "写个API"
- ✅ 好: "使用FastAPI写一个POST端点，接收用户ID和订单数据，返回订单确认信息，需要进行输入验证"

🎯 **原则3: 明确输出格式**
- ❌ 差: "写个计算器"
- ✅ 好: "写一个Python类Calculator，包含add、subtract、multiply、divide方法，每个方法接收两个数字参数并返回结果，需要处理除零错误"

🎯 **原则4: 说明使用场景**
- ❌ 差: "写个循环"
- ✅ 好: "写一个Java方法，用于批量处理用户列表（1000+条），对每个用户执行数据验证和数据库更新，需要考虑性能"

## ✅ 优质Prompt示例

### 示例1: 数据验证函数

**Prompt:**
```
我需要用Python写一个email验证函数。要求：
1. 检查email格式是否正确（基本的@符号和域名检查）
2. 返回布尔值
3. 处理异常情况（None、空字符串）
4. 不使用正则表达式库，用简单字符串操作实现
5. 包含3-5个单行注释说明逻辑

函数名: validate_email
示例：
- validate_email("user@example.com") → True
- validate_email("invalid.email") → False
- validate_email("") → False
```

**预期输出:**
- 清晰的函数定义，包含参数类型提示
- 完整的错误处理逻辑
- 注释清楚
- 可以直接复制使用

**适用场景:** 表单验证、用户注册、数据导入验证

**提示:**
- 可以进一步要求添加单元测试
- 如果需要更复杂的验证，可以用正则表达式替代
- 可以要求支持多种email格式

### 示例2: 数据结构初始化

**Prompt:**
```
用JavaScript写一个Student类，要求：
1. 属性：name (string), age (number), grades (array of numbers), gpa (number)
2. 方法：
   - constructor(name, age): 初始化学生信息，grades和gpa默认为[]和0
   - addGrade(grade): 添加成绩到grades数组，验证范围0-100
   - calculateGPA(): 计算GPA（平均分/25）
   - getInfo(): 返回学生信息摘要

示例使用：
const student = new Student("Alice", 20);
student.addGrade(85);
student.addGrade(92);
console.log(student.calculateGPA()); // 应该输出3.48
```

**预期输出:**
- ES6 class语法
- 包含输入验证
- 每个方法都有简短注释
- 可以直接运行的代码

**适用场景:** 快速构建数据模型、原型开发、学习面向对象编程

### 示例3: 算法实现

**Prompt:**
```
用Go语言实现一个函数，完成以下任务：
任务：给定一个整数数组，找出第一个重复的数字

要求：
1. 函数签名: func findFirstDuplicate(nums []int) int
2. 如果没有重复，返回-1
3. 时间复杂度应尽可能优化（不要用暴力O(n²)）
4. 提供完整的main函数示例，包含3个测试用例

测试用例：
- [1,2,3,4,2,5] → 2
- [1,2,3,4,5] → -1
- [5,5,5,5] → 5
```

**预期输出:**
- 高效的实现（使用map/set）
- 清晰的变量名
- 包含错误处理
- 可运行的完整示例

**适用场景:** 解决编码面试题、学习算法、快速验证想法

### 示例4: Web表单处理

**Prompt:**
```
用React写一个简单的登录表单组件，要求：
1. 字段：username (text input) 和 password (password input)
2. 验证逻辑：
   - username 长度 ≥ 3
   - password 长度 ≥ 6
   - 提交时显示验证错误消息
3. 状态管理用useState Hook
4. 提交按钮在验证失败时应禁用
5. 成功提交时显示"登录成功！"消息

示例：用户输入有效信息后，点击提交按钮应该成功。
```

**预期输出:**
- 完整的函数式组件
- 正确使用useState和事件处理
- 清晰的验证逻辑
- 用户友好的反馈信息

**适用场景:** 快速构建UI组件、学习React基础、原型设计

## ❌ 常见误区

### 误区1: 要求模糊

❌ 不要这样：
```
"给我一个函数"
"写个处理数据的方法"
"做个api端点"
```

✅ 应该这样：
```
"用Python写一个函数，接收CSV文件路径，读取数据并返回清理后的DataFrame"
```

### 误区2: 缺少上下文

❌ 不要：
```
"这段代码有问题，帮我修复"
[只给出代码片段，没有错误信息]
```

✅ 应该：
```
"这段代码在处理大数据集时很慢，错误信息是'MemoryError'，以下是代码...
我预期的性能是处理10万条记录在5秒内完成"
```

### 误区3: 忽视语言和版本

❌ 不要：
```
"写一个async函数处理请求"
[没说是JavaScript还是Python]
```

✅ 应该：
```
"用Python 3.10+写一个async函数，使用aiohttp库发送GET请求..."
```

### 误区4: 不明确约束条件

❌ 不要：
```
"优化这个排序"
```

✅ 应该：
```
"优化这个排序，要求时间复杂度O(nlogn)，空间复杂度不超过O(1)"
```

## 高级技巧

### 技巧1: 使用"思考步骤"模式

对于稍复杂的任务，明确要求AI先思考再编码：

```
请按以下步骤完成：
1. 理解需求（用你自己的话总结一遍）
2. 设计算法（说明关键步骤）
3. 实现代码
4. 提供使用示例

需求：[你的需求]
```

### 技巧2: 明确"好代码"的标准

在Prompt中说明你对代码质量的期望：

```
要求生成的代码：
- 遵循PEP8风格
- 包含类型提示
- 每个函数都有docstring
- 包含错误处理
- 可以直接在生产环境使用
```

### 技巧3: 要求中间过程

```
请不只提供最终代码，还要说明：
1. 解决思路
2. 关键算法的时间复杂度
3. 可能的优化方向
4. 使用示例
```

### 技巧4: 提供反例提升理解

```
我需要一个函数用来排序。
以下是不好的实现（过于复杂）：
[给出一个不好的例子]

请给出一个简洁优雅的实现，并解释为什么更好。
```
