# 🎯 推荐系统设计

> 设计和优化推荐系统的 Prompt 工程实践

---

## 📌 应用场景

为平台或应用设计和优化推荐系统，用于：
- 电商平台（亚马逊、淘宝等）商品推荐
- 流媒体平台（Netflix、B站等）内容推荐
- 社交媒体（抖音、Instagram等）信息流推荐
- 新闻聚合平台（新闻推荐）

---

## 🎯 核心要素

| 要素 | 说明 | 例子 |
|------|------|------|
| **角色** | 推荐系统架构师 | 算法设计、系统优化 |
| **背景** | 业务需求和用户数据 | 平台、用户、内容类型 |
| **目标** | 设计有效的推荐策略 | 提高转化率、用户留存 |
| **约束** | 性能、成本、伦理 | 实时性、冷启动问题 |
| **示例** | 推荐场景和指标 | 点击率、转化率、满意度 |

## ✅ 完整 Prompt 模板

### 基础版（简单推荐场景）

```
你是一位推荐系统架构师。

请根据以下业务信息，设计一个简单但有效的推荐系统。

【业务需求】
- 平台类型: {平台类型：电商/内容/社交等}
- 推荐对象: {推荐什么：商品/视频/文章等}
- 用户规模: {用户数}
- 内容规模: {内容数}
- 主要目标: {目标：提高转化率/增加留存等}

【当前痛点】
- {痛点1}
- {痛点2}
- {痛点3}

【核心约束】
- 性能要求: {实时性要求}
- 成本限制: {预算}
- 技术栈: {现有技术}

【设计要求】
1. 推荐策略框架（4-5个策略）
2. 用户兴趣建模方法
3. 内容相似度计算方式
4. 推荐结果多样性保证
5. 冷启动问题的解决方案
6. 关键指标和监控

【输出格式】
**推荐系统整体设计**

【推荐策略】
1. {策略1名称}：{描述和适用场景}
2. {策略2名称}：{描述和适用场景}
...

【用户建模】
{用户兴趣向量的构建方法}

【内容相似度计算】
{计算方法和公式概述}

【多样性保证】
{如何防止推荐结果同质化}

【冷启动解决方案】
{新用户、新内容的推荐策略}

【关键指标**】
- 点击率 (CTR)：{目标值}
- 转化率 (CVR)：{目标值}
- 推荐多样性：{度量方法}
```

   - 在线层设计（特征服务、实时预测、结果排序）
   - 数据流图（从用户行为到推荐结果）

2. **核心推荐策略**（设计3-5个策略及其权重）
   - 协同过滤（用户-用户 / 用户-内容）
   - 内容推荐（基于属性相似度）

4. **内容建模方案**
   - 内容特征提取（元数据、文本、视觉特征）

   - 内容质量评分（专业评分、用户评分、热度等）
   - 内容新鲜度和时效性考虑

5. **相似度计算和排序**
   - 用户-内容相似度计算方法
   - 内容间相似度计算（用于协同过滤）
   - 多目标排序函数设计
   - 排序模型选择（线性模型/树模型/深度学习）

6. **多样性保证机制**
   - 推荐结果的主题多样性
   - 推荐结果的来源多样性
   - 打破信息茧房的策略
   - 结果去重和相似度控制

7. **冷启动问题解决方案**
   - 新用户冷启动（基于人口统计学特征或随机推荐）
   - 新内容冷启动（基于内容特征的协同过滤）
   - 新兴趣挖掘（A/B测试和探索策略）

8. **实时反馈和学习**
   - 用户反馈信号的收集（点击、收藏、购买等）
   - 反馈信号的权重设定
   - 在线学习和模型更新策略
   - 上下文感知的推荐调整

9. **质量控制和监控**
   - 关键指标定义（CTR、CVR、用户满意度等）
   - 推荐质量评估方法
   - 模型性能监控
   - 异常检测和告警机制
   - A/B测试框架

10. **伦理和可持续性**
    - 推荐公平性（不同用户群体的体验）
    - 隐私保护（个人数据的使用）
    - 推荐的可解释性（用户理解为什么被推荐）
    - 内容多样性和社会责任

## 💡 真实示例

### 示例1：电商推荐系统

**输入信息：**
```
平台：在线电商平台
推荐对象：商品
月活：500万用户
内容库：1000万商品
目标：提高商品点击率和转化率
主要痛点：
1. 冷启动问题严重
2. 推荐结果同质化
3. 个性化程度不够
```

**关键设计决策：**
```
推荐策略权重分配：
- 热销推荐：30% （新用户和冷启动）
- 个性化推荐：40% （基于浏览和购买历史）
- 相似商品推荐：20% （基于当前浏览商品）
- 探索性推荐：10% （发现新兴趣）

用户建模：
- 基于用户浏览历史、购买历史、搜索词等特征
- 用向量表示用户兴趣（维度：商品类别、价格、品牌等）
- 定期更新，最近行为权重更高

冷启动方案：
- 新用户：基于用户人口统计学特征（年龄、地理位置）推荐热销商品
- 新商品：基于商品属性和已有商品反馈推荐

关键指标：
- CTR目标：从3.5% 提升到 5.0%
- CVR目标：从2.0% 提升到 3.0%
- 用户平均会话时长增加20%
```

### 示例2：视频内容推荐系统

**输入信息：**
```
平台：短视频平台
推荐对象：用户生成内容(UGC)视频
月活：2000万用户
内容库：1亿条视频
目标：增加用户留存和日均观看时长
```

**关键设计决策：**
```
推荐策略：
1. 基于观看历史的个性化推荐（60%）
   - 用户观看过的视频的相似内容
   - 用户关注博主的新视频

2. 热门/趋势推荐（20%）
   - 实时热点话题
   - 考虑新鲜度

3. 探索性推荐（15%）
   - 发现用户潜在兴趣
   - A/B测试新话题

4. 社交推荐（5%）
   - 用户粉丝关注的内容

用户建模的特殊性：
- 需要实时捕捉用户兴趣变化（视频平台用户兴趣变化快）
- 短期兴趣（最近观看的视频类型）权重高
- 定期重新训练推荐模型

内容质量评分：
- 播放完成率
- 点赞率和评论率
- 分享率
- 内容审核安全分

关键指标：
- 日均观看时长增加30分钟
- 用户留存率（Day7）从45% 提升到 55%
- 推荐内容的满意度评分 ≥ 4.2
```

## 🔧 优化建议

### ✅ 做的事情

1. **多策略融合**
   ```
   ✅ 很好：结合热销、个性化、相似、探索等多个策略
   ❌ 不好：只用一种推荐算法（容易陷入局部最优）
   ```

2. **平衡精准性和多样性**
   ```
   ✅ 很好：既推荐用户可能喜欢的，也发现新兴趣
   ❌ 不好：过度个性化导致信息茧房
   ```

3. **解决冷启动问题**
   ```
   ✅ 很好：为新用户、新内容提供专门策略
   ❌ 不好：假设所有用户都有充足的历史数据
   ```

4. **持续优化和学习**
   ```
   ✅ 很好：定期A/B测试，根据反馈调整模型
   ❌ 不好：一成不变地使用初版算法
   ```

### ❌ 避免的事情

1. **忽视用户隐私** - 必须遵守隐私法规
2. **过度推荐** - 推荐频率过高导致用户反感
3. **缺乏透明度** - 用户应该理解为什么被推荐
4. **忽视公平性** - 不同用户群体应该有公平的体验
5. **单纯追求短期指标** - 忽视长期用户价值

## 💡 进阶概念

### 协同过滤深度分析
- **User-Based CF**：基于相似用户的推荐
- **Item-Based CF**：基于相似商品的推荐
- **矩阵分解**：用隐因子模型降维

### 深度学习推荐模型
- **DNN推荐**：多层神经网络
- **RNN推荐**：考虑序列特性
- **Attention机制**：突出重要特征
- **GNN推荐**：图神经网络处理用户-内容关系图

### 推荐系统挑战
- **马太效应**：热门内容越来越热，冷门内容无人问津
- **信息茧房**：用户陷入兴趣同质化
- **数据稀疏**：大多数用户-内容对缺乏交互数据
- **实时性**：如何在毫秒级内给出推荐

---

**💡 提示**: 最好的推荐系统不仅能提高转化率，还能让用户发现新的、真正感兴趣的内容，建立长期的用户价值。
