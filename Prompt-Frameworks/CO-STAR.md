# Prompt 框架：CO-STAR

## CO-STAR 框架概述

**CO-STAR** 是一个专为**内容创作和创意输出**设计的 Prompt 框架，特别适合**营销文案、创意写作、广告创意**等场景。框架名称由六个核心要素组成：

- **C**ontext（背景）
- **O**bjective（目标）
- **S**tyle（风格）
- **T**one（语气）
- **A**udience（受众）
- **R**esponse Format（响应格式）

CO-STAR 框架强调**创意和表达风格**，是内容创作者的必备工具。

---

## 特点

| 特点 | 说明 |
|------|------|
| **学习曲线** | 低到中，易于理解 |
| **适用场景** | 内容创作、营销文案、创意写作 |
| **框架复杂度** | ⭐⭐ 中等（6个要素） |
| **有效性** | ⭐⭐⭐⭐⭐ 非常高 |
| **迭代性** | 高（易于迭代调整风格） |
| **个性化** | 强（Style 和 Tone 高度定制化） |

### CO-STAR 的优势

✅ **风格高度可控** - Style 和 Tone 让创意输出更贴近需求
✅ **受众明确** - 针对特定受众的内容效果更好
✅ **易于调整** - 改变 Style 或 Tone 快速迭代
✅ **结构清晰** - 六个要素涵盖内容创作的各个方面
✅ **适合非技术领域** - 不仅程序员，内容团队也常用

### CO-STAR 的局限

❌ **不适合代码生成** - 专注内容创作，代码方面表现一般
❌ **创意难以量化** - "创意好"的标准因人而异
❌ **需要多轮迭代** - 风格调整可能需要反复修改

## 模板

```text
[Context]
背景信息：
- [产品/服务的基本信息]
- [市场环境]
- [竞争情况]
- [重点痛点]

[Objective]
目标：
- [具体的营销目标]
- [期望达成的效果]

[Style]
创作风格：
- [参考的品牌风格]
- [视觉/语言风格方向]
- [例子：简洁/幽默/专业/温暖等]

[Tone]
语气：
- [整体基调：正式/轻松/友好/专业等]
- [避免的语气：...]

[Audience]
受众：
- [目标人群：年龄/职位/兴趣]
- [他们关心什么]
- [他们的痛点]

[Response Format]
输出格式：
- [文案长度：140字/500字等]
- [形式：推文/公众号文章/邮件等]
- [包含元素：标题/副标题/正文/CTA等]
```

## 相关框架比较

| 维度 | CO-STAR | BROKE | CRISPE |
|------|---------|-------|--------|
| **主要用途** | 创意内容 | 代码生成 | 问题分析 |
| **创意性** | 高 | 低 | 中 |
| **可复用性** | 中（风格导向） | 高 | 中 |
| **学习难度** | 低 | 极低 | 中 |
| **迭代次数** | 多（3-5轮） | 少（1-2轮） | 中（2-3轮） |

## 应用实践

### 实践1：产品发布文案

**问题描述**：为新产品发布创作社交媒体文案

```text
[Context]
我们开发了一款新的开源 Java 框架 "FastAPI"，
用于快速构建高性能的 REST API。
竞争对手主要是 Spring Boot（功能全但配置复杂）。
用户痛点：配置繁琐、学习曲线陡、启动慢。

[Objective]
目标：吸引年轻的 Java 开发者尝试和使用这个框架
期望效果：获得 GitHub Star 增长、社区参与

[Style]
参考风格：TensorFlow、Vue.js 等开源项目的文案风格
特点：简洁有力、强调优势而不贬低竞争对手、包含代码示例

[Tone]
整体基调：友好、热情、充满信心
避免：过度吹嘘、复杂术语、枯燥乏味

[Audience]
目标人群：
- Java 开发者，1-10 年经验
- 关心：开发效率、代码简洁度、学习成本
- 痛点：Spring Boot 配置复杂、框架学习时间长

[Response Format]
输出 3 个版本：
1. Twitter 风格（280 字以内）
2. 技术博客摘要（200-300 字）
3. GitHub README 的"为什么选择 FastAPI"部分（500 字）
```

**预期输出**：
- 有吸引力的社交媒体文案
- 突出核心优势的博客摘要
- 有说服力的选择理由

### 实践2：营销邮件

**问题描述**：为 SaaS 产品创作营销邮件

```text
[Context]
我们的产品：企业级数据分析平台 DataViz Pro
目标客户：中型电商公司（100-1000 人）
核心优势：
- 快速可视化任何数据（无需编码）
- 实时数据更新
- 1天部署（不需要 IT 深度参与）

[Objective]
目标：引导潜在客户开始免费试用
期望：提高点击率 > 5%，转化率 > 2%

[Style]
参考企业级产品（Tableau、Looker）的风格
特点：专业、可靠、以数据说话

[Tone]
基调：自信、倾听、以客户需求为中心
避免：过度推销、技术术语堆砌

[Audience]
目标人群：
- 数据分析团队主管 / BI 负责人
- 面临报表生成困难、数据可视化工具成本高
- 重视时间和 ROI

[Response Format]
输出邮件，包含：
- 主题行（50字以内，吸引人）
- 开头钩子（为什么他们应该关心）
- 痛点确认（说出他们的困境）
- 解决方案（我们如何帮助）
- 社会证明（客户案例/数据）
- CTA（清晰的行动召唤）
```

**预期输出**：
- 高转化率的邮件文案
- 打动目标受众的说辞
- 清晰的行动路径

### 实践3：品牌故事

**问题描述**：为公司网站创作品牌故事

```text
[Context]
公司背景：一个 5 人初创公司，做开发者工具
创立时间：2021 年
核心故事：创始人曾是大厂工程师，发现团队开发效率低，自己写了工具

[Objective]
目标：建立品牌认同感，吸引有共鸣的开发者加入社区
期望：让访客理解我们的价值观和使命

[Style]
风格：真实、鲜活、有人情味的创业故事
参考：ProductHunt、Y Combinator 创业公司的品牌故事

[Tone]
基调：真诚、热情、充满使命感
避免：虚伪、过度包装、corporate speak

[Audience]
目标人群：
- 开发者（我们的用户）
- 潜在团队成员（招聘）
- 投资者（融资）

[Response Format]
输出 500-800 字的品牌故事，包含：
- 开头：吸引注意力的场景
- 问题：创始人面临的痛点
- 转折：灵光一现的时刻
- 解决方案：我们做了什么
- 现在：取得的成就
- 未来：我们的愿景
```

**预期输出**：
- 引人入胜的品牌故事
- 建立情感连接的文案
- 体现公司价值观的叙述

## CO-STAR 框架最佳实践

### ✅ 做这些事

1. **明确 Style 和 Tone**
   - 不要假设 AI 知道你想要什么风格
   - 给出具体参考（"像 Apple 的产品文案"）

2. **清楚描述 Audience**
   - 他们是谁、关心什么、痛点是什么
   - 越具体越好

3. **具体的 Response Format**
   - "文案长度 280 字"比"简短文案"更有效
   - "包含 3 个数据点"比"引用数据"更清晰

4. **多轮迭代**
   - 第1轮：AI 给出初稿
   - 第2轮："调整风格，更幽默一些"
   - 第3轮："突出这个特点"

### ❌ 避免这些事

1. **模糊的风格描述**
   - ❌ "文案要好"
   - ✅ "参考 Airbnb 品牌文案的温暖和信任感"

2. **不知道受众是谁**
   - ❌ "写给开发者"（太宽泛）
   - ✅ "初创公司的技术负责人，30-40岁，有10年经验"

3. **没有具体的格式要求**
   - ❌ "写一个文案"
   - ✅ "Twitter 文案，280 字，包含 emoji 和链接"

4. **风格和语气冲突**
   - ❌ Style "专业严肃"，Tone "搞笑"
   - ✅ 风格和语气要协调一致

## 常见问题

**Q: CO-STAR 和 BROKE 的区别是什么？**

A: BROKE 用于代码和需求明确的任务，CO-STAR 用于创意内容和风格可变的任务。

**Q: 可以用 CO-STAR 写代码吗？**

A: 理论上可以，但不推荐。代码需求通常不需要考虑"风格"和"受众"。

**Q: 如何评价 CO-STAR 的输出质量？**

A: 看是否击中目标受众、是否达成营销目标、是否符合品牌风格。
