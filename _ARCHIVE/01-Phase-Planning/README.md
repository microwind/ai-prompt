# 📋 阶段1：需求与规划

> 项目启动、需求分析、架构设计

本阶段的目标是确保项目方向清晰、技术方案合理、资源计划充分。

---

## 📚 本阶段包含的任务

### 01 需求澄清 (Requirement Clarification)
- **难度**: ⭐ 初级
- **何时使用**: 项目启动、需求评审
- **关键能力**: 需求分析、边界确认、范围管理
- **核心问题**: 我们要做什么？为什么要做？

👉 **[打开 01-requirement-clarification/](01-requirement-clarification/)**

---

### 02 架构决策 (Architecture Decision Draft)
- **难度**: ⭐⭐⭐⭐ 中高级
- **何时使用**: 系统设计、技术选型
- **关键能力**: 架构设计、权衡分析、决策记录
- **核心问题**: 怎样设计系统？用什么技术？

👉 **[打开 02-architecture-decision/](02-architecture-decision/)**

---

## 🔄 工作流程

```
收到需求
    ↓
T01: 澄清和确认需求
    ↓
确定系统边界和范围
    ↓
T02: 做架构决策
    ↓
选定技术栈
    ↓
进入下一阶段（设计与开发）
```

---

## ✅ 这个阶段的完成标准

- [ ] 需求被充分理解和确认
- [ ] 项目范围边界清晰
- [ ] 架构方案已制定
- [ ] 技术栈已选定
- [ ] 关键决策已记录
- [ ] 团队达成共识
- [ ] 资源计划已制定

---

## 🎯 最常问的问题

**Q: 需求还在变化，要不要开始设计？**
A: 等待需求稳定。或者设计可调整的架构。

**Q: 怎样快速做架构决策？**
A: 用ADR（架构决策记录）模板，列出方案、优缺点、选择理由。

**Q: 新手能做T01吗？**
A: 完全可以！这是最好的学习机会。
